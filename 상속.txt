상속
  - 부모 Class -> 자식 Class
  - public Class 자식 extends 부모
  - JAVA는 다중상속 불가!, 다중구현 가능
  - 상속을 받은 자식 Class도 반드시 객체 생성 후 사용
  - 자식 인스턴스는 부모의 멤버변수와 멤버함수를 사용
  - 자식 생성자() 최 상단에 super(), 
    super()는 부모생성자를 호출 → 부모 객체 생성
  - 상속받은 자식의 경우 부모객체생성 후 자식객체가 생성 됨
  - 부모가 멤버에 private을 사용하면 상속 X 의미
  - 부모의 멤버 함수를 그대로 사용하지 않고 자식이 재정의해서 사용
    → 오버라이딩
        1. 접근제한자는 부모.멤버함수의 좁은 범위 X
        2. 반환타입, 이름, 매개변수 수정 X
        3. 함수 내용(실행문)만 수정 가능
※ 상속은 코드재사용을 위한 기능 X, 코드 관리 용이성

  - 다형성 개념
    부모 클래스 타입으로 자식 인스턴스를 정의하고 사용!
    매개변수로 값을 전달할 떄 부모 클래스 타입으로
    즉, 다형성으로 코드를 작성하면 향후에 코드 수정없이 자식객체만 추가해서 변경 가능
  - 부모.멤버함수에 final을 사용하면 자식이 오버라이딩 불가

    final 변수: 상수
    final 함수: 오버라이딩 금지
    final 클래스: 상속 금지
